# サンプルアプリ 入力チェック・業務ルール一覧

## 目的

社内向け日報管理システムにおける、入力チェック、業務ルール、権限チェック、状態チェックを定義する。

本資料は、Java / Spring Boot バックエンドのバリデーション実装、React + TypeScript フロントエンドの入力制御、API確認、画面確認、テストケース作成の入力情報として使用する。

## 基本方針

- 入力チェックの最終判定はバックエンドAPIで行う
- フロントエンドの入力制御は、利用者の誤入力を減らすための補助とする
- 入力エラー時は登録、更新、状態変更を行わない
- 入力エラーは画面で確認できるようにする
- APIの入力エラーは `400`、権限エラーは `403`、状態不整合は `409` を基本とする
- 日報登録、編集、提出、再提出では、休日区分ごとの入力チェックを共通で適用する

## 共通形式ルール

| ルールID | 対象 | ルール | エラー時 |
| --- | --- | --- | --- |
| V-COM-001 | 日付 | `yyyy-MM-dd` 形式で扱う | 入力エラー |
| V-COM-002 | 年月 | `yyyy-MM` 形式で扱う | 入力エラー |
| V-COM-003 | 時刻 | 画面・APIでは `HH:mm` 形式で扱う | 入力エラー |
| V-COM-004 | 時間量 | APIとDBでは作業時間、自動算出休憩時間、実勤務時間、通常勤務時間、残業時間、深夜時間を分単位の整数で扱う。画面では時分形式で表示する | 入力エラー |
| V-COM-005 | コード値 | 承認状態、ロール、休日区分コードは定義済み値のみ許可する。休日区分コードは休日区分マスタで管理する | 入力エラー |
| V-COM-006 | 文字列 | 備考、差戻しコメントにカンマ、改行、引用符が含まれてもCSV出力で壊れないように扱う | 実装不備 |
| V-COM-007 | 日付入力 | 画面の日付入力はカレンダー入力を基本とし、手入力前提にしない | 実装不備 |

## 状態変更時の業務ルール

| ルールID | 操作 | ルール |
| --- | --- | --- |
| V-TRN-001 | 提出 | 日報登録と同じ入力チェックを実行する |
| V-TRN-002 | 提出 | 状態を `PENDING` に更新し、提出日時を記録する |
| V-TRN-003 | 承認 | 状態を `APPROVED` に更新し、承認者と承認日時を記録する |
| V-TRN-004 | 差戻し | 差戻しコメントを必須とする |
| V-TRN-005 | 差戻し | 状態を `REJECTED` に更新し、差戻し者、差戻し日時、最新差戻しコメントを記録する |
| V-TRN-006 | 再提出 | 日報登録と同じ入力チェックを実行する |
| V-TRN-007 | 再提出 | 状態を `PENDING` に更新し、提出日時を更新する |
| V-TRN-008 | 再提出 | 最新差戻しコメントは参照可能なまま保持する |
| V-TRN-009 | 状態変更全般 | 状態更新と監査項目更新は同一トランザクションで処理する |
| V-TRN-010 | 状態変更全般 | 更新直前に現在状態を再確認し、同時操作による状態不整合を検出する |

## テスト重点観点

- 休日区分ごとの入力可否
- 休憩区分による休憩時間の自動算出
- 勤務区分による通常勤務時間、残業時間、深夜時間の自動算出
- 実勤務時間と作業時間合計の一致
- 提出、承認、差戻し、再提出の状態遷移
- 権限外データの参照、更新不可
- 承認済み日報のみ月次集計とCSV出力の対象になること
- 差戻しコメントの必須、保持、再提出後の参照
- 同時操作または古い画面操作による状態不整合
