# サンプルアプリ スコープ定義

## 目的

AI活用開発の研究・検討に使用する、社内向け日報管理システムのスコープを定義する。

本サンプルアプリは、ウォーターフォール開発における開発からテストまでの工程を対象に、AIによる実装支援、テスト支援、レビュー支援、工数・品質記録の効果を検証するために作成する。

機能一覧と受入条件は `機能一覧・受入条件.md` を参照する。
画面一覧、画面遷移、主要項目、主要操作は `画面設計.md` を参照する。
API一覧は `API一覧.md` を参照する。
DB概念設計は `DB概念設計.md` を参照する。
状態遷移と業務フローは `状態遷移・業務フロー設計.md` を参照する。
入力チェックと業務ルールは `入力チェック・業務ルール一覧.md` を参照する。
非機能要件は `非機能要件.md` を参照する。
テスト方針と確認観点は `テスト設計書.md` を参照する。

## システム概要

社員が日々の勤務実績と案件別・作業分類別の作業時間を登録し、上長が承認または差戻しを行う。
管理者は全体の月次集計を確認し、CSV出力を行う。

## 利用者ロール

### 社員

- 自分の日報を登録・編集・参照する
- 日報を提出する
- 差戻された日報を修正して再提出する

### 上長

- 承認対象グループの日報を参照する
- 承認対象グループの日報を承認する
- 承認対象グループの日報を差戻す
- 未承認の日報を一覧で確認する

### 管理者

- 全社員の日報を参照する
- 月次集計を確認する
- CSV出力を行う
- 日報の承認・差戻しは行わない

## 対象機能

- ログイン
- 日報登録
- 日報編集
- 日報カレンダー・一覧
- 日報詳細
- 日報提出
- 日報承認
- 日報差戻し
- 日報再提出
- 未承認一覧
- 月次集計
- CSV出力

## 対象外機能

初回サンプル開発では、以下は対象外とする。

- ユーザー追加・変更・削除画面
- グループ追加・変更・削除画面
- 休日区分、休憩区分、案件、作業分類のマスタメンテナンス画面
- 組織変更画面
- パスワード変更
- メール通知
- 承認ルートの多段階化
- 外部システム連携
- スマートフォン専用画面

ユーザー、グループ、上長の承認対象グループ、案件、作業分類、休日区分、休憩区分、勤務区分などのマスタデータは、初期データとして用意する。
初回サンプルでは、利用者は社員、上長、管理者のいずれか1ロールのみを持つものとし、兼務ロールは対象外とする。
初回サンプルでは、社員は1つの所属グループを持つものとし、上長は承認対象グループに所属する社員の日報のみ承認・差戻しできる。
承認対象グループは上長ごとに複数設定できるため、同一社員の日報を複数の上長が承認可能なルートも表現できる。

## 日報状態

- 未提出
- 承認待ち
- 差戻し
- 承認済み

## 承認フロー

```text
未提出
  ↓ 社員が日報を登録・提出
承認待ち
  ↓ 上長が承認
承認済み

承認待ち
  ↓ 上長が差戻し
差戻し
  ↓ 社員が修正・再提出
承認待ち
```

## 状態ごとの操作可否

### 未提出

- 社員は自分の日報を登録・編集・提出できる
- 上長は承認・差戻しできない

### 承認待ち

- 社員は編集できない
- 上長は承認・差戻しできる

### 差戻し

- 社員は自分の日報を編集・再提出できる
- 上長は承認できない

### 承認済み

- 社員は編集できない
- 上長は承認・差戻しできない
- 管理者の月次集計・CSV出力の対象にする

## ロール・状態別の操作可否

| 操作 | 社員 | 上長 | 管理者 |
| --- | --- | --- | --- |
| 自分の日報登録 | 可 | 不可 | 不可 |
| 自分の未提出日報編集 | 可 | 不可 | 不可 |
| 自分の差戻し日報編集 | 可 | 不可 | 不可 |
| 自分の日報提出 | 可 | 不可 | 不可 |
| 自分の差戻し日報再提出 | 可 | 不可 | 不可 |
| 自分の日報参照 | 可 | 不可 | 不可 |
| 承認対象グループの日報参照 | 不可 | 可 | 不可 |
| 全社員の日報参照 | 不可 | 不可 | 可 |
| 承認対象グループの承認待ち日報承認 | 不可 | 可 | 不可 |
| 承認対象グループの承認待ち日報差戻し | 不可 | 可 | 不可 |
| 未承認一覧参照 | 不可 | 可 | 不可 |
| 月次集計参照 | 不可 | 不可 | 可 |
| CSV出力 | 不可 | 不可 | 可 |

管理者は全状態の日報を参照できる。
ただし、管理者は日報の登録、編集、提出、再提出、承認、差戻しを行わない。

### 日報ヘッダ

- 日付

- 勤務開始時刻
- 勤務終了時刻
- 休憩区分
- 休憩時間
- 勤務区分
- 通常勤務時間
- 残業時間
- 深夜時間
- 休日区分
- 備考
- 承認状態
- 差戻しコメント

- 承認者

- 差戻し者

### 作業明細

- 案件
- 作業分類
- 作業時間

1日の日報に複数の作業明細を登録できる。

## 画面導線・日付選択ルール

- 社員の初期画面は、当月の日報カレンダーと当月日報一覧を中心に表示する
- 社員はカレンダーまたは当月日報一覧から対象日付を選択して、日報登録・編集・詳細へ遷移する
- 日報登録・編集の日付入力欄はカレンダー入力を基本とし、手入力を主導線にしない
- 一覧、未承認一覧、月次集計、CSV出力の年月・日付条件もカレンダーまたは年月選択を基本とする
- 社員の日報カレンダー・一覧ではグループ選択を表示しない
- 日報カレンダーでは承認状態の文字をセル内に表示せず、状態色で日付セルを塗り分ける

## 時間入力ルール

- 勤務開始時刻、勤務終了時刻は1分単位で入力する
- 休憩時間は社員に紐づく休憩区分から自動算出し、社員が自由入力しない
- 作業明細の作業時間は分単位で入力する
- 時間計算は分単位で行う
- APIとDBでは、作業時間、休憩時間、実勤務時間、通常勤務時間、残業時間、深夜時間を分単位の整数で扱う
- 画面上の時刻は `HH:mm` 形式で表示する
- 画面上の時間量は `8:00` のような時分形式で表示する

## 時間計算ルール

```text
休憩時間 = 休憩区分に定義された休憩時間帯のうち、勤務時間帯に含まれる時間の合計

実勤務時間 = 勤務終了時刻 - 勤務開始時刻 - 自動算出休憩時間

作業時間合計 = 作業明細の作業時間の合計
```

実勤務時間がある日報では、自動算出休憩時間を加味した実勤務時間と作業時間合計が一致する必要がある。
実勤務時間は、社員に紐づく勤務区分をもとに通常勤務時間、残業時間、深夜時間へ分解して保持・表示する。

## 基本入力チェック

- 社員と日付の組み合わせは一意とし、同一社員・同一日の日報を重複登録できない
- 日付、休日区分は必須とする
- 勤務開始時刻、勤務終了時刻、作業明細の必須可否は休日区分ごとの扱いに従う
- 勤務開始時刻、勤務終了時刻、休憩区分が設定されている場合、実勤務時間が0分以下になる登録は不可
- 自動算出休憩時間が勤務開始から勤務終了までの時間以上になる登録は不可
- 勤務開始時刻、勤務終了時刻が入力されている場合、勤務終了時刻が勤務開始時刻以前になる登録は不可
- 実勤務時間を算出する日報では、社員に勤務区分が設定されていることを必須とする
- 作業明細を登録する場合、作業明細の作業時間は1分以上とする
- 作業明細を登録する場合、作業時間合計は自動算出休憩時間を加味した実勤務時間と一致する必要がある

## 休日区分

- 通常勤務
- 休日
- 有給休暇
- 午前休
- 午後休

休日区分はマスタとして管理し、初回サンプルでは初期データとして用意する。

## 休憩区分

初回サンプルでは、社員に休憩区分を1つ紐づける。
休憩区分は、勤務時間帯に含まれる休憩時間帯をもとに休憩時間を自動算出するために使用する。

休憩区分の例:

| 休憩区分 | 休憩時間帯 |
| --- | --- |
| 標準休憩 | 12:00-13:00 |
| 夕方休憩あり | 12:00-13:00、17:30-17:45 |

休憩区分と休憩時間帯はマスタとして管理し、初回サンプルでは初期データとして用意する。

## 勤務区分

初回サンプルでは、社員に勤務区分を1つ紐づける。
勤務区分は、通常勤務時間帯をもとに実勤務時間を通常勤務時間、残業時間、深夜時間へ分解するために使用する。

勤務区分の例:

| 勤務区分 | 通常勤務時間帯 |
| --- | --- |
| 勤務区分A | 09:00-18:00 |
| 勤務区分B | 09:00-17:30 |

深夜時間帯は初回サンプルでは全勤務区分共通で `22:00-05:00` とする。
勤務区分はマスタとして管理し、初回サンプルでは初期データとして用意する。

## 休日区分ごとの扱い

### 通常勤務

- 勤務開始時刻、勤務終了時刻、作業明細が必要
- 休憩時間は休憩区分から自動算出する
- 作業明細は1件以上必要
- 実勤務時間と作業時間合計が一致する必要がある

### 休日

- 作業なしでも登録可能
- 休日出勤として作業時間の登録も可能
- 作業明細がない場合は、勤務開始時刻、勤務終了時刻を入力しない
- 作業明細がある場合は、勤務開始時刻、勤務終了時刻が必要
- 作業明細がある場合、休憩時間は休憩区分から自動算出する
- 作業明細がある場合は、実勤務時間と作業時間合計が一致する必要がある

### 有給休暇

- 勤務開始時刻、勤務終了時刻、休憩時間は登録不可
- 作業明細は登録不可

### 午前休

- 午後の勤務として、勤務開始時刻、勤務終了時刻、作業明細が必要
- 休憩時間は休憩区分から自動算出する
- 作業明細は1件以上必要
- 実勤務時間と作業時間合計が一致する必要がある

### 午後休

- 午前の勤務として、勤務開始時刻、勤務終了時刻、作業明細が必要
- 休憩時間は休憩区分から自動算出する
- 作業明細は1件以上必要
- 実勤務時間と作業時間合計が一致する必要がある

## 差戻しコメントの扱い

- 差戻し時は差戻しコメントを必須とする
- 日報ヘッダには最新の差戻しコメントを保持する
- 再提出後も最新の差戻しコメントは参照可能とする
- 初回サンプルでは、差戻しコメントの履歴管理は対象外とする

## 月次集計

管理者は、月単位で日報の集計を確認できる。

初回サンプルでは、以下の集計を対象とする。

- 社員別作業時間
- 案件別作業時間
- 作業分類別作業時間
- 休日区分別日数

承認済みの日報を月次集計の対象とする。

## CSV出力

管理者は、月次集計または日報明細をCSV出力できる。

初回サンプルでは、CSV形式を以下に固定する。

- 文字コード: UTF-8 BOM付き
- 改行コード: CRLF
- 区切り文字: カンマ
- 囲み文字: ダブルクォート
- エスケープ: 値にダブルクォートを含む場合、ダブルクォートを2つ重ねる
- 日付形式: `yyyy-MM-dd`
- 時刻形式: `HH:mm`
- 時間量形式: 分単位の整数
- 月次集計CSVファイル名: `monthly-summary-{yyyyMM}.csv`
- 日報明細CSVファイル名: `daily-report-details-{yyyyMM}.csv`

月次集計CSVは、社員別作業時間、案件別作業時間、作業分類別作業時間、休日区分別日数を同一形式で出力する。
`aggregate_type` には `EMPLOYEE_WORK`、`PROJECT_WORK`、`CATEGORY_WORK`、`HOLIDAY_TYPE_DAYS` のいずれかを出力する。
該当しない列は空欄とする。

月次集計CSVのヘッダー:

```text
year_month,aggregate_type,employee_id,employee_name,project_id,project_name,work_category_id,work_category_name,holiday_type,total_work_minutes,total_days
```

日報明細CSVのヘッダー:

```text
report_date,employee_id,employee_name,holiday_type,start_time,end_time,break_minutes,project_id,project_name,work_category_id,work_category_name,work_minutes,approval_status,remarks
```

日報明細CSVは作業明細単位で出力する。
作業明細がない休日・有給休暇の日報は1日1行で出力し、案件、作業分類は空欄、作業時間は0分として扱う。

## 完了条件

対象機能について、以下を満たすこと。

- 実装が仕様を満たしている
- 人間レビューが完了している
- 単体テストが成功している
- 主要機能はAPI確認と画面確認が完了している
- 提出、承認、差戻し、再提出、月次集計、CSV出力はE2E相当の業務フロー確認が完了している
- 主要な異常系、境界値、権限、状態遷移が確認されている
- AI生成物の採用判断が記録されている
