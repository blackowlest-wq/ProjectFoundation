# サンプルアプリ 画面設計

## 目的

社内向け日報管理システムの画面一覧、画面遷移、主要項目、主要操作を定義する。

本資料は、AIに実装やテスト作成を依頼する際の入力情報として使用する。
詳細な入力チェック、権限、状態遷移、CSV形式は以下を参照する。

- スコープ定義: `スコープ定義.md`
- 機能一覧・受入条件: `機能一覧・受入条件.md`
- API一覧: `API一覧.md`
- 状態遷移・業務フロー設計: `状態遷移・業務フロー設計.md`
- 入力チェック・業務ルール一覧: `入力チェック・業務ルール一覧.md`
- 非機能要件: `非機能要件.md`
- テスト設計書: `テスト設計書.md`
- UIモック: `UIモック/index.html`

## 画面遷移

```text
ログイン画面
  ↓ ログイン成功
日報カレンダー・一覧画面

日報カレンダー・一覧画面
  → 日報登録画面
  → 日報詳細画面

日報登録画面
  → 登録
  → 登録して提出
  → 日報カレンダー・一覧画面

日報詳細画面
  → 日報編集画面
  → 提出
  → 承認
  → 差戻し入力ダイアログ
  → 日報カレンダー・一覧画面

日報編集画面
  → 保存
  → 再提出
  → 日報詳細画面

未承認一覧画面
  → 日報詳細画面

月次集計画面
  → CSV出力画面

CSV出力画面
  → CSV出力
```

## S-001 ログイン画面

### 目的

利用者がログインし、ロールに応じた初期画面へ遷移する。

### 主要項目

- ログインID
- パスワード
- エラーメッセージ

### 主要操作

- ログイン

### 遷移

- ログイン成功時は日報カレンダー・一覧画面へ遷移する
- ログイン失敗時はログイン画面に留まり、エラーを表示する

## S-002 日報カレンダー・一覧画面

### 目的

権限に応じて日報をカレンダーまたは一覧で確認し、対象日付の日報へ遷移する。

### 主要項目

検索条件:

- 対象年月
- 日付範囲
- 社員
- 承認状態
- 休日区分

カレンダー項目:

- 日付
- 休日区分
- 承認状態色
- 実勤務時間
- 差戻し有無

一覧項目:

- 日付
- グループ
- 社員名
- 休日区分
- 勤務時間
- 作業時間合計
- 承認状態
- 提出日時
- 承認者
- 承認日時

### 主要操作

- 検索
- 条件クリア
- カレンダーの日付を選択する
- 日報詳細を開く
- 日報登録画面へ遷移する

### 表示制御

- 社員は自分の当月日報を初期表示し、月カレンダーと当月日報一覧を表示する
- 社員は日付を手入力せず、カレンダーまたは当月日報一覧から登録・編集・詳細へ遷移できる
- 社員向け表示では、グループのプルダウン検索条件を表示しない
- カレンダーは承認状態を文字で表示せず、日付セルの背景色または左罫線色で表現する
- 上長は承認対象グループの日報のみ表示する
- 管理者は全社員の日報を表示する
- 社員のみ日報登録画面への遷移を表示する

## S-003 日報登録画面

### 目的

社員が自分の日報を新規登録する。

### 主要項目

日報ヘッダ:

- 日付
- 休日区分
- 勤務開始時刻
- 勤務終了時刻
- 備考

情報表示:

- 社員名
- 所属グループ
- 休憩区分
- 勤務区分
- 自動算出休憩時間
- 実勤務時間
- 通常勤務時間
- 残業時間
- 深夜時間
- 作業時間合計

作業明細:

- 案件
- 作業分類
- 作業時間

### 主要操作

- 作業明細を追加する
- 作業明細を削除する
- 保存する
- 保存して提出する
- 一覧へ戻る

### 表示制御

- 日付はカレンダー入力とする
- 社員本人が登録する場合、社員名は情報表示とする。上長または管理者が参照・代理確認する画面では、権限範囲内の社員を選択可能にする
- 通常勤務、午前休、午後休では勤務時刻、作業明細を入力可能にする
- 休日では、作業明細なし登録と休日出勤登録の両方を許可する
- 有給休暇では、勤務時刻、休憩時間、作業明細を登録不可にする
- 休憩区分、勤務区分は社員に紐づく値を情報表示し、登録画面では編集不可にする
- 自動算出休憩時間、実勤務時間、通常勤務時間、残業時間、深夜時間、作業時間合計を画面上に時分形式で表示する

## S-004 日報編集画面

### 目的

社員が未提出または差戻しの日報を編集する。

### 主要項目

日報登録画面と同じ項目を表示する。

追加表示:

- 承認状態
- 最新差戻しコメント
- 差戻し者
- 差戻し日時

### 主要操作

- 作業明細を追加する
- 作業明細を削除する
- 保存する
- 再提出する
- 詳細へ戻る

### 表示制御

- 未提出の日報は保存と提出を可能にする
- 差戻しの日報は保存と再提出を可能にする
- 承認待ち、承認済みの日報は編集画面を利用できない

## S-005 日報詳細画面

### 目的

日報の内容、承認状態、監査情報を参照し、状態に応じた操作を行う。

### 主要項目

日報ヘッダ:

- 日付
- 社員名
- 所属グループ
- 休日区分
- 勤務開始時刻
- 勤務終了時刻
- 休憩区分
- 休憩時間
- 勤務区分
- 実勤務時間
- 通常勤務時間
- 残業時間
- 深夜時間
- 作業時間合計
- 備考
- 承認状態

作業明細:

- 案件
- 作業分類
- 作業時間

監査情報:

- 提出日時
- 承認者
- 承認日時
- 差戻し者
- 差戻し日時
- 最新差戻しコメント

### 主要操作

- 編集する
- 提出する
- 承認する
- 差戻しする
- 一覧へ戻る

### 表示制御

- 社員は自分の日報のみ参照できる
- 上長は承認対象グループの日報のみ参照できる
- 管理者は全社員の日報を参照できる
- 社員は未提出の日報を提出できる
- 社員は未提出または差戻しの日報を編集できる
- 上長は承認待ちの日報を承認・差戻しできる
- 管理者は承認・差戻し操作を利用できない

## S-006 未承認一覧画面

### 目的

上長が承認対象グループの承認待ち日報を一覧で確認する。

### 主要項目

検索条件:

- 日付範囲
- グループ
- 社員

一覧項目:

- 日付
- グループ
- 社員名
- 休日区分
- 勤務時間
- 作業時間合計
- 提出日時

### 主要操作

- 検索
- 条件クリア
- 日報詳細を開く

### 表示制御

- 承認待ちの日報のみ表示する
- 承認対象グループ以外の日報は表示しない

## S-007 差戻し入力ダイアログ

### 目的

上長が差戻しコメントを入力して日報を差戻す。

### 主要項目

- 差戻しコメント
- エラーメッセージ

### 主要操作

- 差戻しを確定する
- キャンセルする

### 表示制御

- 差戻しコメントは必須とする
- 承認待ちの日報に対してのみ表示する

## S-008 月次集計画面

### 目的

管理者が月単位で作業時間と休日区分日数を確認する。

### 主要項目

検索条件:

- 対象年月

集計表示:

- 社員別作業時間
- 案件別作業時間
- 作業分類別作業時間
- 休日区分別日数

### 主要操作

- 集計表示
- CSV出力画面へ遷移する

### 表示制御

- 承認済み日報のみ集計対象にする
- 社員と上長は利用できない

## S-009 CSV出力画面

### 目的

管理者が月次集計CSVまたは日報明細CSVを出力する。

### 主要項目

- 対象年月
- 出力種別
  - 月次集計CSV
  - 日報明細CSV

### 主要操作

- CSV出力
- 月次集計画面へ戻る

### 表示制御

- 管理者のみ利用できる
- CSV形式は `スコープ定義.md` のCSV出力仕様に従う

## 画面設計上の補足

- 画面文言、細かなレイアウト、色、アイコンは本資料の対象外とし、画面構成と表示優先度は `UIモック/index.html` を参照する
- 入力チェックの詳細は `入力チェック・業務ルール一覧.md` に従う
- 画面確認では、権限ごとの表示制御、状態ごとの操作可否、休日区分ごとの入力制御を重点確認する
