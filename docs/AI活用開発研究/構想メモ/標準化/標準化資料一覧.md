# 標準化資料一覧

## 目的

標準化フォルダ内の資料を探しやすくし、作業内容に応じて確認すべき資料を選べるようにする。

## 最初に確認する資料

| 資料 | 用途 |
| --- | --- |
| `AGENTS.md` | AI作業時の入口、記録ルール、開発標準資料への導線 |
| `開発フロー.md` | IntakeからRecordまでの標準フロー |
| `プロンプトテンプレート集.md` | フェーズ別にAIへ依頼するための正本テンプレート |

## 実装前に使う資料

| 資料 | 用途 |
| --- | --- |
| `実装前チェック表.md` | 作業ごとの確認結果を埋めるチェック表 |
| `実装前確認観点.md` | 実装前確認のチーム共通判断基準 |
| `ディレクトリ構成ルール.md` | ファイル配置、責務分離、フォルダ判断 |
| `セキュリティ規約.md` | 認証、認可、CSRF、入力検証、ログ出力 |
| `共通部品化判断基準.md` | 共通化、重複排除、過剰共通化の判断 |

## 退避した旧資料

次の資料は役割が重複したため、正本へ必要な内容を統合し、`旧資料/` 配下へ退避した。
新規作業では参照しない。

| 旧資料 | 統合先 |
| --- | --- |
| `旧資料/AI向けプロンプト集_実運用版.md` | `プロンプトテンプレート集.md` |
| `旧資料/AI活用開発手順書.md` | `開発フロー.md` |
| `旧資料/AI生成物レビュー観点表.md` | `実装後レビュー観点.md` |

## 運用ルール

- 新しい資料を追加した場合は、この一覧にも用途を追加する。
- 既存資料と役割が重なる場合は、どちらを正本として使うかを明記する。
- 資料を参照できなかった場合は、理由と代替確認方法を作業記録に残す。
