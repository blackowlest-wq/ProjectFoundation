# AI活用開発手順書

## 目的

開発者が迷わずAIを使えるように、開発からテストまでの工程におけるAI活用手順を標準化する。

本手順は、社内向け日報管理システムのサンプル開発を対象とする。
人間が設計し、AIが実装・テストを支援する前提で運用する。
サンプルアプリのスコープは `../../サンプル設計書/スコープ定義.md` を参照する。

## 前提

- 開発モデル: ウォーターフォール
- 対象工程: 開発からテストまで
- フロントエンド: React + TypeScript
- バックエンド: Java / Spring Boot
- データベース: Oracle Database
- AIの位置づけ: 開発者を支援する補助者
- 最終判断者: 人間の開発者またはレビュー担当者

## 基本ルール

- AIに設計判断を丸投げしない
- AIに渡す前に、入力情報を整理する
- AI生成物は人間レビューなしで採用しない
- AIが仕様外の判断をした場合は、実装せず確認事項として扱う
- AI利用に伴う確認・修正時間も作業実績に含める

## 標準フロー

```text
1. 入力情報を準備する
2. AIに実装方針を確認させる
3. 仕様曖昧箇所を洗い出す
4. AIにコード生成を依頼する
5. 人間が生成物を確認・修正する
6. AIにテスト観点を洗い出させる
7. AIにテストコードまたはテストケースを生成させる
8. テストを実行する
9. 不具合があればAIに原因候補と修正案を出させる
10. 人間が修正内容と回帰テストを確認する
```

## 工程別手順

### 1. 実装前

目的:

- 仕様理解のズレを減らす
- 実装前に曖昧な仕様を検出する
- テスト観点を先に確認する

入力情報:

- 対象機能の仕様
- 画面項目
- API仕様
- DBテーブル定義
- 権限・状態遷移
- 入力チェック条件
- 受入条件

AIへの依頼:

- 実装方針の確認
- 仕様曖昧箇所の洗い出し
- 実装リスクの洗い出し
- テスト観点の抽出

人間の確認:

- AIの指摘に仕様根拠があるか
- 仕様外の判断をしていないか
- 実装前に確認すべき事項が残っていないか

成果物:

- 実装方針メモ
- 仕様確認事項
- テスト観点メモ

### 2. 実装中

目的:

- 実装作業を効率化する
- 入力チェック、例外処理、権限制御の漏れを減らす
- 既存ルールに沿ったコードを生成する

入力情報:

- 実装対象機能
- 対象ファイルまたは関連コード
- コーディング規約
- API仕様
- DB定義
- テスト観点

AIへの依頼:

- フロントエンド実装
- バックエンドAPI実装
- バリデーション実装
- DBアクセス実装
- エラーハンドリング実装
- 既存コードとの整合性確認

人間の確認:

- 仕様通りに実装されているか
- 不要な処理や過剰な抽象化がないか
- 認証・認可が考慮されているか
- 入力チェックとエラーメッセージが妥当か
- DB更新やトランザクションが妥当か

成果物:

- 実装コード
- 修正メモ
- AI利用ログ

### 3. テスト作成

目的:

- 正常系だけでなく異常系・境界値・権限・状態遷移を補強する
- テスト作成作業を効率化する

入力情報:

- 対象機能の仕様
- 実装コード
- 受入条件
- 統合担当が `メトリクスと分類.md` で行う観点分類・欠陥分類の整理結果

AIへの依頼:

- 単体テストコード生成
- APIテストケース生成
- 画面テストシナリオ生成
- E2Eテストシナリオ生成
- テスト観点の不足確認

人間の確認:

- テストが仕様に基づいているか
- 正常系に偏っていないか
- 異常系、境界値、権限、状態遷移が含まれているか
- テストデータが現実的か
- テストが壊れやすい実装依存になっていないか

成果物:

- 単体テスト
- APIテスト
- E2Eテストシナリオ
- テスト観点メモ

### 4. 不具合修正

目的:

- 原因調査と影響範囲確認を効率化する
- 修正漏れと回帰漏れを減らす

入力情報:

- 不具合内容
- 再現手順
- エラーログ
- 関連コード
- 関連仕様
- 既存テスト結果

AIへの依頼:

- 原因候補の洗い出し
- 影響範囲の確認
- 修正案の提示
- 回帰テスト観点の提示

人間の確認:

- 原因が再現可能か
- 修正案が仕様に合っているか
- 副作用がないか
- 回帰テストが不足していないか

成果物:

- 修正コード
- 原因メモ
- 影響範囲メモ
- 回帰テスト

## AI利用ログ

最低限、以下を記録する。

```text
日付:
対象機能:
利用工程:
AIへの依頼内容:
AI出力の採用有無:
採用した内容:
修正した内容:
不採用理由:
確認者:
```

## 完了条件

対象機能について、以下を満たした場合に開発完了とする。

- 実装が仕様を満たしている
- 人間レビューが完了している
- 単体テストが成功している
- 主要機能はAPI確認と画面確認が完了している
- 承認、差戻し、月次集計、CSV出力はE2E相当の業務フロー確認が完了している
- 主要な異常系、境界値、権限、状態遷移が確認されている
- AI生成物の採用判断が記録されている
