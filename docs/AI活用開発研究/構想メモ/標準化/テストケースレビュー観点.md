# テストケースレビュー観点

## 目的

単体テスト、API/Controllerテスト、フロント単体テスト、E2E、Oracle/DB接続テストをレビューするときに、仕様担保、境界値、実装分岐、品質補完の抜け漏れを確認するための共通チェックリストを定義する。

## 使い方

1. 対象機能の受入条件、入力チェック、業務ルール、非機能要件を確認する。
2. 既存テストケースを下記チェックリストに対応付ける。
3. 判定は `OK`、`不足`、`対象外`、`保留` のいずれかにする。
4. `不足` は追加候補として記録し、仕様上必要か、カバレッジ目的か、回帰防止目的かを分類する。

## 基本観点

| 観点 | 確認内容 |
| --- | --- |
| ブラックボックス | 仕様書、API一覧、受入条件どおりの入力と出力を確認しているか |
| ホワイトボックス | 実装上の重要な分岐、例外、状態変化を確認しているか |
| 正常系 | 代表的な成功パターンを確認しているか |
| 異常系 | 不正入力、未認証、権限不足、存在しないデータを確認しているか |
| 最小値 | 入力可能な最小値、最小件数を確認しているか |
| 最大値 | 許容最大値、最大文字数、上限件数を確認しているか |
| 境界値 | 境界値そのものと境界の外側を確認しているか |
| 同値分割 | 同じ扱いになる入力群から代表値を選んでいるか |
| 区分値・Enum網羅 | 定義済み区分を全件確認しているか。未定義値を拒否しているか |
| 組み合わせ | ロール、状態、区分、入力有無などの組み合わせを確認しているか |
| デシジョンテーブル | 条件が多い業務ルールを表にして漏れなく確認しているか |
| 状態遷移 | 正常遷移、不正遷移、遷移後の副作用を確認しているか |
| null・未指定 | null、未指定、空配列などを確認しているか |
| 空文字・空白 | 空文字、半角空白、全角空白を確認しているか |
| 形式チェック | 日付、時刻、年月、コード形式などを確認しているか |
| 桁数・精度 | 文字数、数値桁、丸め、分単位などを確認しているか |
| 重複・一意制約 | 同じキーや業務上一意のデータを拒否できるか |
| 存在チェック | 参照先データや対象データが存在しない場合を確認しているか |
| 権限・所有者 | 自分、他人、上長、管理者など権限差を確認しているか |
| 順序・並び替え | 一覧や検索結果の順序を確認しているか |
| 件数 | 0件、1件、複数件、大量件数を確認しているか |
| 複数データ読み込み | 複数レコードや複数ロールの読み込みで期待どおりか |
| N+1 | データ件数増加時にSQLや処理回数が不自然に増えないか |
| 排他・同時実行 | 同時更新、古い状態、二重実行を確認しているか |
| 副作用 | DB更新、セッション、ログ、イベントなどの副作用を確認しているか |
| 冪等性 | 同じ操作を複数回実行しても破綻しないか |
| タイムゾーン・日付境界 | 月末、年末、日跨ぎ、タイムゾーン差異を確認しているか |
| ロケール・文字種 | 日本語、全角、記号、改行、絵文字などを確認しているか |
| セキュリティ | 認証、認可、CSRF、XSS、SQLインジェクションを考慮しているか |
| エラー処理 | 期待するステータス、エラーコード、メッセージ、詳細を返すか |
| DB依存 | 実DB、方言、制約、型差異を確認しているか |
| 回帰テスト | 過去の不具合やレビュー指摘の再発を防ぐテストがあるか |
| カバレッジ補完 | 未通過分岐を意図して確認しているか。仕様担保との違いを明記しているか |

## 区分値・Enumがある場合の重点観点

| 観点 | 確認内容 |
| --- | --- |
| 定義済み値の全件確認 | Enumや区分値を全件1回以上確認しているか |
| 未定義値の拒否 | 存在しない区分値、空、nullを拒否しているか |
| 区分ごとの業務差分 | 区分によって結果や権限が変わる箇所を確認しているか |
| 区分と状態の組み合わせ | 区分 × 状態 × 操作の許可/拒否を確認しているか |
| 追加時の変更容易性 | Enum追加時にテストが不足を検知できるか |

## レビュー結果の記録形式

| 観点 | 判定 | 現行テスト | 不足・追加候補 | 優先度 |
| --- | --- | --- | --- | --- |
| 例: 境界値 | 不足 | 81文字ログインIDのみ | 80文字OK、100文字パスワードOK、101文字パスワードNG | Medium |
