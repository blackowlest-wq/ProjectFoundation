# ProjectFoundation Review

## 目的

AI実装後のレビュー観点を揃え、指摘修正と記録更新を漏らさない。

## 参照する資料

- `AGENTS.md`
- `docs/AI活用開発研究/構想メモ/標準化/開発フロー.md`
- `docs/AI活用開発研究/構想メモ/標準化/実装後レビュー表.md`
- `docs/AI活用開発研究/構想メモ/標準化/テスト・静的解析チェック表.md`
- `docs/AI活用開発研究/構想メモ/標準化/サブエージェントプロンプト集.md`
- 対象機能の作業記録、指摘一覧、テストケース資料

## 手順

1. 差分と対象機能を確認する。
2. 仕様、受入条件、入力チェック、状態遷移、DB、認証認可、セキュリティを確認する。
3. テスト不足、カバレッジ未通過、E2E不足を確認する。
4. 可読性、ファイル配置、命名、コメント、共通化を確認する。
5. 必要に応じてサブエージェント用プロンプトを使い、観点別レビューを依頼する。
6. 指摘を重要度、分類、対応状況に分ける。
7. 修正したらテストを再実行する。
8. 作業記録、指摘一覧、レビュー記録を更新する。

## 指摘出力形式

```text
指摘ID:
分類:
重要度:
対象:
内容:
根拠:
修正案:
対応状況:
対応内容:
保留理由:
再確認条件:
```

## 完了条件

- 重大な仕様不整合、認証認可、CSRF、DB整合性、テスト不足が確認済みである。
- 必要なテストが成功している、または未実行理由が記録されている。
- 指摘一覧と作業記録が更新されている。
