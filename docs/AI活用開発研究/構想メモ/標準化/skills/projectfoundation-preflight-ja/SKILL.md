# ProjectFoundation Preflight

## 目的

実装前に必要な情報を揃え、PlanなしでDoに進むことを防ぐ。

## 参照する資料

- `AGENTS.md`
- `docs/AI活用開発研究/構想メモ/標準化/開発フロー.md`
- `docs/AI活用開発研究/構想メモ/標準化/実装前チェック表.md`
- `docs/AI活用開発研究/構想メモ/標準化/プロンプトテンプレート集.md`
- 対象機能に関係する設計書
- 既存の作業記録、指摘一覧、レビュー記録

## 手順

1. まだコード変更しない。
2. 対象機能、対象外、完了条件を整理する。
3. 設計書から、画面、API、DB、入力チェック、状態遷移、非機能、テスト設計を抽出する。
4. 既存実装から、ログイン、共通API、CSRF、Oracle接続、テスト方針、ファイル配置を確認する。
5. 受入条件をBDDシナリオに分解する。
6. 業務ルール、入力チェック、状態遷移、計算ロジックのTDD対象を決める。
7. 必要なテスト、静的解析、品質ゲートを決める。
8. 確認事項、仮定、保留事項を分離する。

## 出力形式

```text
対象機能:
実装対象:
対象外:
参照した設計書:
既存実装で合わせる箇所:
BDDシナリオ:
TDD対象:
必要なAPI:
必要なDB/DDL/マスタ:
必要な画面:
必要なテスト:
静的解析/品質ゲート:
確認事項:
仮定:
保留事項:
次のDoで守ること:
```
