# 日報登録・編集画面 指摘一覧

## 目的

ここしばらくの日報登録・編集画面に対するレビュー、共通化検討、DBマスタ化検討で発生した指摘を、作業記録とは別に追跡できるよう一覧化する。

作業記録は「何をしたか」を残す資料、指摘一覧は「何を問題として検出し、どう扱ったか」を残す資料として分ける。

## 運用ルール

- レビューで問題を検出した場合は、作業記録だけでなくこの指摘一覧にも追記する。
- 対応済みにする場合は、対応ファイルまたは対応記録先を必ず記載する。
- 保留にする場合は、保留理由と再確認タイミングを記載する。
- 同じ問題を複数資料に書く場合でも、指摘IDはこの一覧を正とする。
