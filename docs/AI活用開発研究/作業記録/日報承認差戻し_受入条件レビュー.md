## 対象外・保留

### 対象外

- 差戻しコメントの履歴・通知・承認ルート多段階化
- 管理者の承認・差戻し
- 楽観ロック・競合更新の原子性保証
- 月次集計、CSV出力、保守画面

### 保留と再確認条件

- `rejectComment` の前後空白を保存値にも反映するか: DTO/既存入力規約確認時
- A-009の監査フィールドのnull表現とJSON名: DTO設計時
- エラー本文の詳細フィールド、409後の再取得・表示文言: 既存共通エラー契約と画面設計確認時
- 監査日時の表示精度: API DTOと既存日時シリアライズ確認時
- Oracle実行可否: 保護されたOracle TEST環境とrunnerが利用可能になった時点

## レビュー結果

- F-005/F-007/F-008/F-010の受入条件は、TC-APR-001～010へ少なくとも1件ずつ追跡できる。
- API、UI、DB、副作用なし、権限、状態、コメント境界、CSRF、E2Eの観測点をケースまたは保留欄へ配置した。
- 設計書にない期待値（具体的なエラー文言、DBの内部実装、日時の丸め、コメント履歴）は追加していない。
- ケースレビュー上の指摘は発行していないため、`FIND-APR-*` は未発行である。実装後に検出した指摘は作業記録と指摘一覧へ反映する。
