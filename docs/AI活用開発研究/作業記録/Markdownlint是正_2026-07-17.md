# Markdownlint是正作業記録

## 作業日

2026-07-17

## 依頼内容

リポジトリ全体のMarkdownlintエラーを確認し、設定不備とMarkdown本文の規約違反を解消する。

## 検出内容

- 初回実行では、Markdownlintのglobがローカルツール、エージェント状態、worktree、依存物を十分に除外できず、プロジェクト資料以外のMarkdownも検査対象になっていた。
- Git管理対象を中心としたプロジェクトMarkdownでは、重複見出し、見出し・リスト・表の前後空行、表の列区切り、コードフェンス言語の違反を検出した。
- 設計書やサンプル資料では、異なる親見出しの下で同じ子見出しを繰り返す構造があり、単純な重複見出し禁止では文書構造と合わなかった。

## 対応内容

- `.markdownlint-cli2.jsonc` に `gitignore: true` を設定し、`.gitignore` で管理対象外としているローカルツール、worktree、依存物、生成物を検査対象から除外した。
- Markdownの対象globを `**/*.md` に整理した。
- `MD024` に `siblings_only: true` を設定し、異なる親見出し配下の再利用見出しを許容した。同一階層の重複は引き続き検出する。
- Markdownlintの自動修正で空行、末尾空白、表周辺の整形を修正した。
- `標準化/作業記録テンプレート.md` の空セルをcompact形式へ整え、末尾のコードフェンスを `text` として明示した。

## 確認結果

- 初回のプロジェクトMarkdown検査: 80ファイル、223件。
- 自動修正後の残存: 44件。作業記録テンプレートの表整形とコードフェンスだけに限定された。
- 作業記録追加前の `npm.cmd run lint:markdown`: 80ファイル、0件。
- 作業記録追加後の最終 `npm.cmd run lint:markdown`: 81ファイル、0件。
- `git diff --check`: 成功。
- コード変更はなく、アプリケーションの単体テスト・結合テスト・E2Eは未実行とした。

## 次回確認事項

- 新しいMarkdown資料を追加した場合は、通常の `npm.cmd run lint:markdown` で全体確認する。
- ローカル専用資料や生成物を追加する場合は、`.gitignore` とMarkdownlintの検査対象が一致していることを確認する。
