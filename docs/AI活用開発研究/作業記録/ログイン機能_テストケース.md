# ログイン機能 テストケース

## 対象

- 機能ID: F-001 ログイン
- API: A-001 `/api/auth/login`, A-002 `/api/auth/logout`, A-003 `/api/auth/me`
- 画面: S-001 ログイン画面、ログイン後のロール別初期表示

## テスト方針
- バックエンド通常テストは Oracle 実機を利用する。
- Cookie セッションを利用するため、ログイン後の同一セッション利用、ログアウト後の無効化、CSRF保護を確認する。
- 入力チェックはバックエンドで担保し、フロントエンドはロール別初期遷移ロジックとログイン入力の事前チェックを確認する。

## Oracle 実機接続テスト

| テストID | テスト名 | 対象 | 前提 | 操作 | 期待結果 | 対応する受入条件・要件 |
| --- | --- | --- | --- | --- | --- | --- |
| BE-ORA-001 | Oracle Databaseでログインできる | `AuthControllerOracleIT.loginSucceedsWithOracleDatabase` | Oracle `ORCL` に `users` テーブルが作成済み。`DAILY_REPORT_DB_USER` と `DAILY_REPORT_DB_PASSWORD` を設定済み | `oracle-test` プロファイルで `POST /api/auth/login`、`GET /api/auth/me`、CSRF付き `POST /api/auth/logout` を実行 | Oracle JDBC接続でログインは `200`。`/me` は `employee001` を返す。ログアウトは `204` | DBはOracle Databaseを使用する。Oracle実機でログイン機能が動作する |

実行例:

```powershell
cd backend
$env:DAILY_REPORT_DB_URL='jdbc:oracle:thin:@//localhost:1521/ORCL'
$env:DAILY_REPORT_DB_USER='DAILY_REPORT_TEST'
$env:DAILY_REPORT_DB_PASSWORD='<password>'
$env:SESSION_COOKIE_SECURE='false'
mvn test
```

## カバレッジ取得

### バックエンド

JaCoCo Maven Plugin を利用する。

```powershell
cd backend
mvn -Pcoverage test
```

出力先:

```text
backend/target/site/jacoco/index.html
backend/target/site/jacoco/jacoco.xml
```

### フロントエンド

Vitest coverage を利用する。`@vitest/coverage-v8` が未導入の場合は、先に以下を追加する。

```powershell
cd frontend
npm install -D @vitest/coverage-v8
```

コンソールで確認する場合は以下を実行する。

```powershell
cd frontend
npm run coverage
```

HTMLで確認する場合は以下を実行する。

```powershell
cd frontend
npm run coverage:html
```

出力先:

```text
frontend/coverage/index.html
frontend/coverage/lcov.info
```
