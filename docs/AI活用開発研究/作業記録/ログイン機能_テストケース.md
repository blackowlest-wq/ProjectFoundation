# ログイン機能 テストケース

## 対象

- 機能ID: F-001 ログイン
- API: A-001 `/api/auth/login`, A-002 `/api/auth/logout`, A-003 `/api/auth/me`
- 画面: S-001 ログイン画面、ログイン後のロール別初期表示

## テスト方針

- バックエンド通常テストは Oracle 実機を利用する。
- Cookie セッションを利用するため、ログイン後の同一セッション利用、ログアウト後の無効化、CSRF保護を確認する。
- 入力チェックはバックエンドで担保し、フロントエンドはロール別初期遷移ロジックとログイン入力の事前チェックを確認する。
