# 日報登録・編集画面 単体テストケースレビュー

## 目的

ログイン機能の単体テストケースレビュー観点を使い、日報登録・編集画面のテストケース不足を確認し、必要な追加・修正を行う。

## 観点別レビュー

| 観点 | 判定 | 対応 |
| --- | --- | --- |
| ブラックボックス | 対応 | 通常勤務登録、休日、有給、重複、権限、CSRF、状態不整合、404/403、再提出を確認 |
| ホワイトボックス | 対応 | 休憩自動計算、勤務時間内訳、保存済み日報再検証、深夜/残業分岐を確認 |
| 正常系 | 対応 | 登録、詳細取得、編集、提出を分割 |
| 異常系 | 対応 | 作業時間不一致、重複、権限外、未ログイン、CSRF なし、時刻形式、休憩超過、保存済み不整合 |
| 最小値・最大値 | 一部不足 | 作業時間 1 分境界、備考 1000 文字境界は今後追加候補 |
| 境界値 | 一部不足 | 時刻逆転、休憩時間が勤務時間を超える境界は確認済み。備考長は今後追加候補 |
| null・未指定 | 対応 | 休日・有給で勤務時刻なし、休日区分未指定、勤務時刻未指定を確認 |
| 空文字・空白 | 未対応 | API レイヤでは時刻空文字が未指定扱い。文字列項目の空白は今後追加候補 |
| Enum 網羅 | 一部不足 | WORKDAY、HOLIDAY、PAID_LEAVE を確認。AM_OFF、PM_OFF は今後追加候補 |
| 状態遷移 | 対応 | DRAFT から PENDING、PENDING 編集不可を確認 |
| 権限・所有者 | 対応 | 未ログイン、上長登録不可、社員本人操作を確認 |
| セキュリティ | 対応 | CSRF なし登録が 403 になることを追加 |
| エラー処理 | 対応 | 400、401、403、409 の代表ケースを確認 |
| DB 依存 | 対応 | Oracle 実機で通常テストと JaCoCo を実行済み |

## 残課題

- AM_OFF、PM_OFF の API テスト
- 作業時間 1 分、備考 1000/1001 文字の境界値
- 他社員の日報参照・編集不可。現状は複数社員の初期データが不足しているため保留
- Oracle 固有のトランザクション分離やロック競合の追加確認
