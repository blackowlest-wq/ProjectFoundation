# 利用者向けREADME整備作業記録

## 作業日

2026-07-17

## 依頼内容

AI利用ルール・注意事項.mdの要否を見直し、利用者が既存資料の入口、利用方法、資料間の関係を把握できるREADMEを整備する。

## 対応内容

- ルートREADME.mdを、AIに実装を依頼する利用者と通常の開発者の双方が利用できる入口として再構成した。
- AI実装依頼、実装前確認、テストケースレビュー、品質ゲート、作業記録の利用順をREADMEに整理した。
- AI内部プロンプト、標準化資料、作業記録・研究資料、退避済み旧資料の利用区分をREADMEに記載した。
- AI利用ルール・注意事項.mdを削除し、機密情報を入力しないこと、人による採用判断、仕様不明点の確認、生成物の事実確認、レビュー指摘の記録という要点をREADME、AGENTS.md、AI実装依頼テンプレートで扱う構成にした。
- 標準化資料一覧.mdから削除した資料の行を除去した。
- `docs/superpowers/plans/` と `docs/superpowers/specs/` は中間成果物のため削除済みであり、READMEおよび本記録から現行の参照先として扱わないよう整理した。

## 利用状況の整理

- AI利用ルール・注意事項.mdは、標準化資料一覧から参照されるだけで、現行の作業手順からの参照が少ない資料だったため、独立資料としては削除した。
- `旧資料/`配下は現行資料へ統合済みのため、新規作業の入口ではない。
- 作業記録、検証記録、構想メモは、現行作業の入口ではなく、判断履歴・検証証跡・再利用知見の確認に利用する。
- AI内部プロンプトと標準化資料内のAI用スキルは、利用者が直接編集する資料ではなく、AIの実行定義として管理する。
