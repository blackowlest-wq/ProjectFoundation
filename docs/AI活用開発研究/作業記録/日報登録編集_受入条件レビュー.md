# 日報登録・編集画面 受入条件レビュー

## 目的

`機能一覧・受入条件.md` のうち、今回の実装対象である F-002、F-003、F-006、F-009 の日報登録・編集画面関連範囲について、既存テストケースに不足がないか確認する。

## 判定サマリ

| 機能 | 判定 | 主な不足 |
| --- | --- | --- |
| F-002 日報登録 | 一部不足 | 複数作業明細、作業時間1分、休日作業あり、AM_OFF/PM_OFF、画面確認 |
| F-003 日報編集 | 一部不足 | 差戻し編集、承認済み編集不可、編集時重複、他社員編集不可、画面確認 |
| F-006 日報提出 | 一部不足 | 差戻し/承認済み提出不可、他社員提出不可、提出E2E相当の画面確認 |
| F-009 日報再提出 | 一部不足 | 提出日時更新、差戻しコメント保持、PENDING/APPROVED再提出不可、他社員再提出不可、再提出E2E相当の画面確認 |

## F-002 日報登録

| 受入条件 | 現状 | 対応テスト | 不足 |
| --- | --- | --- | --- |
| 通常勤務を登録できる | 対応 | `createWorkdayReportCalculatesMinutes` | なし |
| 休憩時間を自動算出 | 対応 | `createWorkdayReportCalculatesMinutes`, `TimeRulesTest` | なし |
| 実勤務、通常、残業、深夜を自動算出 | 一部対応 | `createWorkdayReportCalculatesMinutes`, `validateAndCalculateSplitsShortWorkTypeEveningBreakOvertimeAndNight` | APIレスポンスで残業・深夜の確認が薄い |
| 実勤務時間と作業時間合計一致で登録 | 対応 | `createWorkdayReportCalculatesMinutes` | なし |
| 作業明細を複数件登録できる | 不足 | なし | 複数明細の登録・表示順・合計確認が必要 |
| 作業時間は1分単位で登録できる | 一部対応 | 0分拒否は確認済み | 1分登録成功の境界値が必要 |
| 休日かつ作業明細なしで登録 | 対応 | `createAllowsHolidayWithoutWorkItemsAndPaidLeave` | なし |
| 休日かつ作業明細ありで登録 | 不足 | なし | 休日作業あり、時刻あり、合計一致の登録成功が必要 |
| 有給休暇を勤務入力なしで登録 | 対応 | `createAllowsHolidayWithoutWorkItemsAndPaidLeave` | なし |
| 重複登録できない | 対応 | `createRejectsDuplicateReportForSameEmployeeAndDate` | なし |
| 終了時刻が開始時刻以前なら登録不可 | 対応 | `createRejectsEndTimeBeforeStartTime` | なし |
| 休憩時間が勤務時間以上なら登録不可 | 対応 | `createRejectsBreakLongerThanElapsedTime` | なし |
| 作業明細合計不一致なら登録不可 | 対応 | `createRejectsWorkItemTotalMismatch` | なし |
| 作業明細が必要な休日区分で0件不可 | 対応 | `createRejectsWorkdayMissingTimesAndItems` | AM_OFF/PM_OFF は未確認 |
| 有給で勤務時間または作業明細あり不可 | 対応 | `createRejectsPaidLeaveWithWorkInput` | なし |
| 休日作業なしで勤務時刻あり不可 | 対応 | `createRejectsHolidayWithoutItemsButWithTimes` | なし |
| 社員は他社員の日報を登録できない | 一部対応 | `managerCannotCreateEmployeeReport` | APIに社員指定項目がない前提は確認済みだが、別社員データでの明示確認は不足 |
| 登録後状態は未提出 | 対応 | `createWorkdayReportCalculatesMinutes` | なし |
| API確認と画面確認 | 一部不足 | API、フロント補助関数、APIクライアントは確認済み | 画面コンポーネント操作またはE2E相当確認が不足 |

## F-003 日報編集

| 受入条件 | 現状 | 対応テスト | 不足 |
| --- | --- | --- | --- |
| 未提出の日報を編集できる | 対応 | `editDraftReportKeepsDraftStatus` | なし |
| 差戻しの日報を編集できる | 不足 | なし | REJECTED の編集成功が必要 |
| 編集時も登録と同じ入力チェック | 一部対応 | 登録側入力チェック多数 | PUTでの入力チェック代表ケースが不足 |
| 承認待ちは編集不可 | 対応 | `pendingReportCannotBeEdited` | なし |
| 承認済みは編集不可 | 不足 | なし | APPROVED の編集不可が必要 |
| 他社員の日報を編集できない | 一部不足 | `updateMissingOwnReportReturnsForbidden` | 実在する他社員日報への編集不可が必要 |
| 未提出編集後も未提出 | 対応 | `editDraftReportKeepsDraftStatus` | なし |
| 差戻し編集後も差戻し | 不足 | なし | REJECTED 編集後に REJECTED 維持を確認 |
| 編集しても同一社員・同一日が重複しない | 不足 | なし | 別日の日報を既存日へ変更する PUT 重複確認が必要 |
| API確認と画面確認 | 一部不足 | APIクライアントのPUTは確認済み | 画面コンポーネント操作またはE2E相当確認が不足 |

## F-009 日報再提出

| 受入条件 | 現状 | 対応テスト | 不足 |
| --- | --- | --- | --- |
| 差戻しの日報を修正できる | 不足 | なし | REJECTED 編集成功が必要 |
| 差戻しの日報を再提出できる | 対応 | `rejectedReportCanBeResubmitted` | なし |
| 再提出後は承認待ち | 対応 | `rejectedReportCanBeResubmitted` | なし |
| 再提出後、提出日時が更新される | 一部不足 | `submittedAt` 非nullのみ確認 | 既存提出日時から更新されたことの確認が必要 |
| 最新差戻しコメントを参照できる | 不足 | なし | reject_comment を持つ REJECTED を再提出後に詳細取得して保持確認が必要 |
| 再提出時に登録と同じ入力チェック | 一部対応 | `TimeRulesTest` | resubmit APIでの保存済み不整合確認が不足 |
| 入力違反日報は再提出不可 | 不足 | なし | resubmit APIで400確認が必要 |
| 未提出、承認待ち、承認済みは再提出不可 | 一部不足 | DRAFT は `resubmitDraftReportIsRejected` | PENDING、APPROVED が不足 |
| 他社員の日報を再提出できない | 不足 | なし | 実在する他社員日報への再提出不可が必要 |
| 再提出後の日報は編集不可 | 一部対応 | PENDING編集不可は確認済み | 再提出した日報での編集不可確認は不足 |
| API確認と画面確認 | 一部不足 | APIは一部確認済み | 画面操作確認が不足 |
| 提出、差戻し、修正、再提出までのE2E相当 | 不足 | なし | 差戻しAPI未実装のため、現時点では完全E2E不可 |

## 共通受入条件

| 受入条件 | 現状 | 不足 |
| --- | --- | --- |
| 権限のない利用者は対象機能を利用できない | 一部対応 | 他社員データに対する参照・編集・提出・再提出の明示テスト不足 |
| 入力エラー時は登録・更新・状態変更が行われない | 一部対応 | レスポンスステータスは確認済み。DB未更新の明示確認は一部不足 |
| エラー内容は画面で確認できる | 一部不足 | APIエラー変換は確認済み。画面表示としての確認が不足 |
| API確認と画面確認の両方 | 一部不足 | API中心。React画面コンポーネントまたはE2E相当の操作テストが不足 |
| E2E相当の業務フロー確認 | 一部不足 | 登録→提出はAPIで確認済み。差戻し・再提出の完全フローは承認/差戻し未実装のため不足 |
| 同一社員・同一日の日報は重複登録できない | 対応 | 登録重複は確認済み。編集重複は不足 |

## 優先して追加すべきケース

| 優先度 | ケース |
| --- | --- |
| High | F-003: REJECTED 編集成功、REJECTED 編集後状態維持 |
| High | F-003: APPROVED 編集不可 |
| High | F-003: 編集時の日付重複拒否 |
| High | F-006/F-009: 他社員の日報を提出・再提出できない |
| High | F-009: reject_comment 保持、submittedAt 更新確認 |
| Medium | F-002: 複数作業明細登録、休日作業あり登録、1分境界 |
| Medium | F-002: AM_OFF、PM_OFF の登録成功・入力チェック |
| Medium | 画面コンポーネントまたはE2E相当の登録・編集・提出・再提出確認 |

## 対象外として扱う受入条件

以下は今回の「日報登録・編集画面」実装範囲外のため、未充足だが今回の不足ケースには含めない。

- F-004 日報カレンダー・一覧
- F-005 日報詳細画面全体。ただし編集画面から詳細取得する API の一部は確認済み
- F-007 日報承認
- F-008 日報差戻し
- F-010 未承認一覧
- F-011 月次集計
- F-012 CSV出力
