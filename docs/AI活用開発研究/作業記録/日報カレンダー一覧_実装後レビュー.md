## 確認済み観点

- `GET /api/daily-reports` は `SecurityConfig` の `anyRequest().authenticated()` により未ログインを拒否する構成。
- 変更系APIではないため、一覧検索API自体にCSRFは不要。
- 一覧APIはバックエンドで対象期間必須、開始日 > 終了日の検証を行っている。
- 社員は自分の日報のみ、上長は `manager_group_permissions` の担当グループのみ、管理者は全体から検索する実装になっている。
- フロント API 呼び出しは既存 `getJson` / `postJsonWithCsrf` / `putJsonWithCsrf` を使っており、API client 共通化方針に概ね沿っている。
- ファイル配置は `dailyReport` 機能配下、バックエンドは Controller / Service / DTO / Entity / Repository の既存配置に沿っている。
- 作業記録、テストケース資料、指摘一覧の更新は本レビューで実施した。

## 残るリスク

- 個人情報、認証、ロール別参照制御が関係するため、通常の実装後レビューとは別に「セキュリティレビュー依頼」を実施する必要がある。
- バックエンドの実DB接続テストは Oracle 資格情報未設定により未完了だったため、`backend/config/oracle-test.properties` 設定後に再実行が必要。
- 上長・管理者の画面操作E2Eは未整備。API側のロール別テストは追加されているが、実DB接続での実行は未完了。
