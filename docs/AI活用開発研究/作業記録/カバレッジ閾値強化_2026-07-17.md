- `PrePush`は、push対象コミットの空白、生成物、Gitleaks、差分Frontend ESLint、差分Markdownlint、Java変更時のaggregator Spotlessを実行する。
- Frontend設定変更時はFrontend lint全体を実行し、Frontendソース変更時は変更ファイルへESLintを限定する。
- Frontend unit / typecheck / build、Backend test / Checkstyle / SpotBugs、coverageは`Full`とrequired CIに残した。

### 判断理由

単純に`pre-push`の`Full`を`Quick`へ置き換えると、Quickがstaged対象を前提とするため、push済みコミットの差分を正しく検査できない。
そのため、push標準入力からrefとSHAを解析する専用Modeを設けた。PrePushはローカルの待ち時間を短縮するための早期検査であり、
最終判定を担うPR / mainのrequired CIを置き換えない。

### 検証

- `scripts/pre-push.tests.ps1`: ref解析、削除ref、新規branch、必須定義、Full検査の非包含を契約テストで確認した。
- PrePush通常更新: 明示入力と標準入力の両方で成功した。
- PrePush新規branch: remote SHAがゼロの入力で成功した。
- Lefthook実行: `use_stdin: true`を含む`lefthook run pre-push`で成功した。
- PrePushの実測は環境依存だが、今回のローカル検証では約12～16秒だった。Fullのテスト・build・静的解析はCIで継続する。
