# ログイン機能 作業記録

## 対象

- 機能ID: F-001 ログイン
- API: A-001 `/api/auth/login`, A-002 `/api/auth/logout`, A-003 `/api/auth/me`
- 画面: S-001 ログイン画面、ログイン後のロール別初期画面

## 工数

| 区分 | 工数 |
| --- | ---: |
| 実装工数 | 3.2h |
| テスト作成工数 | 1.6h |
| 環境構築・DB・テストデータ準備工数 | 0.4h |

## 実装内容

- Spring Boot バックエンドを新規作成
- Cookie セッション方式のログイン、ログアウト、ログイン中利用者取得 API を実装
- 初期ユーザーとして社員、上長、管理者を登録
- パスワードは BCrypt ハッシュで保存
- 未ログイン状態で保護 API にアクセスした場合は `401` を返す
- React + TypeScript のログイン画面、セッション復元、ログアウト処理、ロール別初期表示を実装
- Oracle `ORCL` 向けの接続プロファイル、`users` テーブルDDL、作成・削除SQLを追加
- 共通DB接続設定を環境変数で上書き可能にし、デフォルト接続先を `ORCL` に更新
- ログアウトAPIをCSRF保護対象に戻し、フロントエンドからCSRFトークンを送信するよう修正
- ログイン成功時にセッションIDを再発行するよう修正
- 本番相当のCookie Secure属性、DDL検証、DB資格情報の外部化を反映
- 可読性レビュー指摘に基づき、フロントエンドコンポーネント、ログインセッション確立処理、初期データ生成、例外レスポンス生成を分割
- バリデーションメッセージとテストヘルパーを整理し、テスト名と検証内容を一致させた
- 単体テストケースレビューで検出した不足観点を指摘記録に追加し、優先度の高いバックエンドテストを追加
- ファイル名・フォルダ構成レビューで検出した不要script、TypeScript対象外設定、`.mjs`混在、DTO集約を修正
- ログインID・パスワードを半角英数字のみ許容する入力チェックをバックエンドとフロントエンドに追加

## テスト内容

- 有効なログインID、パスワードでログインできる
- `/api/auth/me` がログイン中利用者を返す
- 無効なパスワードでは `401` を返す
- 未ログインの `/api/auth/me` は `401` を返す
- ログアウト後は `/api/auth/me` が `401` になる
- ロール別初期パスのマッピングを確認
- Oracle 実機接続で `employee001` のログインと `/api/auth/me` を確認
- CSRFトークンなしのログアウトが `403` になる
- CSRFトークンありのログアウトが成功する
- 長すぎるログインIDが `400` になる
- 可読性改善後も通常テスト、Oracle実機テスト、フロントエンドテスト、ビルドが成功する
- 全ロールのログイン成功、存在しないログインID、未指定・空文字・空白、ログインID/パスワード境界値、セッションID再発行、パスワードハッシュ非露出、日本語ユーザー名を確認
- Vitestを導入し、TypeScriptの `authApi.ts` を直接単体テストできるようにした
- フロントエンドでログインAPI呼び出し、401処理、APIエラー処理、CSRFヘッダー付与を確認
- ファイル名・フォルダ構成修正後にバックエンドテスト、フロントエンドテスト、型チェック、ビルド、HTMLカバレッジ生成を再実行
- 半角英数字以外のログインID・パスワードが入力エラーになることを確認

## 手戻り内容

- MockMvc テストで `JSESSIONID` Cookie を直接検証していたが、MockMvc では `MockHttpSession` を利用する形に修正
- Oracle 実機向けDDLと接続プロファイルが未整備だったため追加
- 初期ユーザーデータの日本語文字列が文字化けしていたため修正し、OracleテストDBのデータを入れ直し
- セキュリティレビュー指摘としてCSRF、セッション固定、Cookie Secure、DB資格情報、DDL自動更新、入力長制限を修正
- 可読性レビュー指摘として責務集中、長い三項演算子、位置引数依存、固定バリデーションメッセージ、テスト名不一致を修正
- 単体テストケースレビュー指摘として境界値、区分値、入力未指定、セッションID再発行、機密情報非露出の不足を修正
- ファイル名・フォルダ構成レビュー指摘としてVitest移行前script残存、型チェック対象漏れ、`.mjs`混在、DTOファイル名不一致を修正

## 発生したエラー

- MockMvc テストで `JSESSIONID` Cookie が取得できず、ログイン後のセッション引き継ぎ確認が失敗
- Oracleテストユーザーへの権限付与前に `ORA-01045` が発生したため、接続手順に必要な権限付与SQLを追記
- セキュリティレビューでログアウトCSRF除外、セッションID未更新、DB資格情報直書きなどの不備を検出
- 可読性レビューでコード理解コストが高い箇所を検出
- 単体テストケースレビューで境界値内側、存在しないログインID、全ロールAPI応答、セッションID再発行、パスワードハッシュ非露出の不足を検出
- ファイル名・フォルダ構成レビューで不要script、TypeScript対象外設定、`.mjs`混在、DTO集約を検出

## AI出力を人間が修正した内容

- 初期ユーザーデータの文字化けについて、AI出力後の確認で修正対象として記録
- 可読性レビューの観点はユーザー提示のリーダブルコードチェックリストに基づいて整理

## 採用判断

- 設計書の範囲に合わせ、パスワード変更、アカウントロック、多要素認証は対象外
- Oracle 接続設定を標準設定に置き、バックエンド通常テストも Oracle 実機を利用する構成に変更
- CSRF は Cookie セッション要件に合わせて有効化し、ログイン API のみ除外した
- バックエンドとフロントエンドのテスト成功結果を採用する
- テストケースは `ログイン機能_テストケース.md` に文章化した
- カバレッジ取得はバックエンドを JaCoCo、フロントエンドを Vitest coverage で行う
