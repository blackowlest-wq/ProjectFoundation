# 日報登録・編集画面 セルフレビュー

## レビュー結果

| 観点 | 結果 | メモ |
| --- | --- | --- |
| 設計書との整合性 | 一部保留 | 登録・編集・提出・再提出の画面/API 範囲は対応。承認、差戻し、一覧、集計、CSV は対象外として保留 |
| 受入条件との整合性 | 一部保留 | F-002、F-003 の主要条件を実装。F-006、F-009 は画面から呼ぶ API 範囲を実装 |
| 入力チェック | 対応 | 休日区分別、時刻順、作業時間 1 分以上、実勤務時間と作業明細合計一致をバックエンドに実装 |
| 状態遷移 | 対応 | DRAFT の提出、REJECTED の再提出、DRAFT/REJECTED の編集可否を実装 |
| DB 設計 | 一部保留 | 日報ヘッダ・明細テーブルを追加。マスタ正規化は既存構成に合わせ固定マスタで代替 |
| 認証・認可 | 対応 | 認証 API 以外は既存 SecurityConfig により認証必須。登録・編集・提出は社員本人のみ |
| CSRF・セッション | 対応 | 更新系 API は既存 CSRF 設定対象。フロント API は `XSRF-TOKEN` を `X-XSRF-TOKEN` で送信 |
| 例外処理 | 対応 | `ApiException` を共通エラーレスポンス形式に統合 |
| 可読性 | 対応 | 計算ロジックを `TimeRules` に分離 |
| ファイル構成 | 対応 | 既存 `auth` と同じ薄い Controller/Service/Repository 構成 |
| テスト妥当性 | 対応 | フロント、Oracle 実機接続のバックエンド通常テストを実行済み |
| カバレッジ | 対応 | フロント coverage と JaCoCo coverage を取得済み。バックエンド Lines 97.77% |

## セキュリティ確認

- 未ログインアクセスは既存 SecurityConfig により 401
- 権限外ロールの登録・編集・提出は 403
- 他社員編集は、社員本人の `employeeUserId` とログイン中ユーザーで照合
- パスワードやハッシュは日報 API レスポンスに含めない
- 更新系 API は CSRF 保護対象

## 指摘・保留

- 設計書の完全な DB 概念設計に合わせるには、マスタテーブルと初期データ投入への拡張が必要
- 一覧画面から登録・編集へ遷移する導線は未実装のため、現在は社員初期画面に直接フォームを表示している
- Oracle 固有のロック競合や同時更新は追加確認候補
