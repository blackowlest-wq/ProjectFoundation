# ドキュメント入口整備 設計

## 目的

ProjectFoundation を初めて参照する人が、プロジェクトの概要と既存ドキュメントの正本へ迷わず到達できる入口を整える。

## 対象範囲

- リポジトリ直下に日本語の `README.md` を追加する。
- README から既存の標準化資料と作業記録へリンクする。
- 既存の標準化資料・作業記録の構成と内容は変更しない。

## 対象外

- `.gitignore` の拡張。
- ソースコード、依存関係、起動設定の変更。
- 詳細な開発手順・アーキテクチャ文書の新設。
- ライセンスの選定。
- 将来の複数バージョン構成の定義。

## 設計

`README.md` はリポジトリの唯一の簡潔な入口とする。掲載内容は、日報管理アプリであること、現行構成がフロントエンド（React/Vite）・バックエンド（Spring Boot）・Oracle Database であること、詳細なルールと履歴は既存文書を参照することに限定する。

開発ルールの正本は `docs/AI活用開発研究/構想メモ/標準化/標準化資料一覧.md` とする。README には個別の規約を転記せず、この一覧へのリンクを設ける。作業の履歴とレビュー指摘は `docs/AI活用開発研究/作業記録/` を参照するよう案内する。

将来のバージョン分割に関する構成、起動方法の詳細、ライセンスは、内容が確定してから別途追加する。今回のREADMEには推測による方針や未確認の運用を記載しない。

## 成功条件

- リポジトリ直下のREADMEから、標準化資料一覧と作業記録へ移動できる。
- 既存の規約を複製せず、正本が一意である。
- 複数バージョンの将来設計を前提とする記載を追加しない。
